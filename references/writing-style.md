# Reference: 中文写作风格

## 目标

让中文写作（文章、笔记、说明、润色）读起来像 lailai 写的：简洁、自然、专业，理科骨架，没有 AI 腔。提炼自站点 `docs/project/guides/text.mdx`（lailai 的《文本格式指南》）与 `.claude/rules/writing-style.md`。

## 适用场景

- 写或润色中文技术文本、文章、笔记、说明。
- 把 AI 生成的中文改写成 lailai 风格。
- 评审一段中文是否"像 lailai"。

> 用词细节见 [wording.md](wording.md)；纯排版见 [markdown-style.md](markdown-style.md)；公式见 [latex-math-style.md](latex-math-style.md)；Docusaurus 站内规范见 [docusaurus-style.md](docusaurus-style.md)。

## 核心规则

### 语气与风格

1. 语言风格**简洁、自然、专业**。理解、干、偶尔自嘲式冷幽默，短陈述句为主，不用感叹号堆情绪。
2. **不要套话开场**：不写"本文将……""众所周知"。直接进入主题。（注：站点文档的 frontmatter `description` 允许"本文为……"，那是元数据，不是正文。）
3. **不要套话收尾**：不写"希望本文对你有所帮助""总而言之，让我们……"。技术内容以复杂度 / 代码 / 结论自然收尾，不加"总结"。
4. **先结论后依据**：判断、结论、取舍放前面。
5. **理科骨架**：定义 → 分类 → 推导 → 例子。

### 句子（强约束）

6. **每句避免超过 40 字，不应超过 60 字**。长难句拆短。
7. 用**简单句和并列句**，避免复合句。
8. 用**主动语态**，避免被动（"程序会自动保存文件"，不是"文件会被程序自动保存"）。
9. 避免双重否定。

### 用词（强约束）

10. 用词遵循 [wording.md](wording.md)。**避免「显然」「易得」「不难发现」**。
11. 避免不必要的**形容词**和**语气词**；避免不必要的「的」「了」「着」等虚词。
12. 避免**网络流行语、谐音错别字、Emoji、颜文字**；避免冷僻、生造、文言文词语；用现代汉语常用表达。
13. 代词指代清晰，不产生歧义。
14. 避免不必要的英文单词（"这个想法很好"，不是"这个 idea 很好"）。

### 术语与翻译

15. 首次出现专业术语用 `中文全称（原文全称，缩写）`：「快速傅里叶变换（Fast Fourier Transform，FFT）」。
16. 首次出现外国人名用 `中文译名（原文姓名，生卒年）`：「艾萨克·牛顿（Isaac Newton，1643–1727）」。
17. 翻译遵循信达雅；专有名词大小写正确（YouTube、GitHub、iPhone）。

### 严谨

18. 不确定的信息标注出来；不编来源、不编数据。重要内容给"参考资料"。
19. 用 GenAI 辅助生成的文本须经人工审核（尤其可能被误认为本人发言时，见 [boundaries](../profile/boundaries.md)）。

## 推荐做法

- 用"是什么 / 为什么 / 怎么做 / 例子"组织一段解释。
- 一个抽象规则后跟一个具体例子。
- 列举互斥情况时分点写清，而不是揉成一段。
- 措辞直接：能说"用 X"就不说"我们可以考虑使用 X 来实现"。

## 不推荐做法

- "这是一个很好的问题""让我们深入探讨"等套话。
- 排比 + 抒情 + 升华的"作文体"；形容词堆叠。
- 「显然」「易得」「不难发现」跳过推理。
- 长难句、被动语态、双重否定。
- 把简单结论藏在三段铺垫之后。

## 示例

**改写前（AI 腔）：**

> 在当今快速发展的技术世界中，前缀和是一个非常重要且强大的概念。本文将带你深入浅出地理解它，希望能对你有所帮助！

**改写后（lailai 风格）：**

> **前缀和**（Prefix Sum）把数列前 $k$ 项和预处理出来，使区间求和从 $O(n)$ 降到 $O(1)$。
>
> 定义 $S_k=\sum_{i=1}^{k}a_i$，则区间 $[l,r]$ 的和为 $S_r-S_{l-1}$。

## Self-review Checklist

- [ ] 简洁、自然、专业，没有套话开场 / 收尾？
- [ ] 先结论后依据，理科骨架（定义 / 分类 / 推导 / 例子）？
- [ ] 每句 ≤ 40 字（最多 60），主动语态，无复合句 / 双重否定？
- [ ] 用词过了 [wording.md](wording.md)，没有「显然 / 易得 / 不难发现」？
- [ ] 没有多余形容词 / 语气词 / 虚词，没有 Emoji / 网络流行语？
- [ ] 术语用 `中文（English，缩写）`，人名带原文与生卒年？
- [ ] 不确定信息已标注，没有编造？
- [ ] 对照 [ai-tone-blacklist.md](ai-tone-blacklist.md) 过了一遍？
