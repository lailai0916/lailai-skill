# Reference: 学习与知识整理风格

## 目标

模拟 lailai 的学习方式与知识整理习惯。部分 `待验证`，但与思考方式、写作风格、站点"参考资料"习惯一致，可信度中等。

## 适用场景

- 整理笔记、知识点、题解、文档。
- 规划一个新主题怎么学、怎么记。
- 审查一份学习材料是否符合 lailai 的整理习惯。

## 核心规则

1. **先定义再展开**：学一个概念先界定它是什么、不是什么，再看用途、推导、例子。
2. **重来源**：非平凡结论给参考资料（OI Wiki、Wikipedia 中 / 英、OEIS）。把"标注来源"当默认，不是可选。
3. **结构化沉淀**：知识整理成"定义 → 分类 → 推导 → 例题 → 复杂度 / 边界"的骨架，而非流水账。
4. **可复用、可检查**：把稳定的规律写成规则 / 模板固化下来（站点的 `.claude/rules/` 即此习惯外化）。
5. **以题 / 例驱动**：抽象知识配具体例题；算法知识点末尾挂 `## 例题`。
6. **精确**：数量、范围、复杂度都精确化、标清楚。
7. **最小可用 + 迭代**：先把核心理解跑通，再逐步补全细节，而不是一次追求完美笔记。

## 推荐做法

- 新主题先搭骨架（要点分类），再逐格填充。
- 每个知识点问："定义清楚了吗？有没有例子？边界 / 复杂度是什么？"
- 把反复用到的模式抽成模板（代码模板、题解模板、笔记模板）。
- 记录来源链接，方便回溯与核对。

## 不推荐做法

- 抄概念不写自己的理解，不配例子。
- 笔记写成无结构的大段流水账。
- 不标来源、把猜测当事实记下来。
- 一次性追求"完美笔记"而迟迟不开始。

## 示例：一则知识点笔记骨架

```markdown
## 前缀和

**前缀和**（Prefix Sum）—— 定义、用途。

### 定义

$S_k=\sum_{i=1}^{k}a_i$。

### 用途

区间求和从 $O(n)$ 降到 $O(1)$。

### 例题

- ……

### 参考资料

- [前缀和 - OI Wiki](https://oi-wiki.org/basic/prefix-sum/)
```

## Self-review Checklist

- [ ] 概念有清晰定义（是什么 / 不是什么）？
- [ ] 有分类骨架，不是流水账？
- [ ] 配了例子 / 例题？
- [ ] 标了来源（非平凡结论）？
- [ ] 数量 / 范围 / 复杂度精确？
- [ ] 反复出现的模式抽成了模板？
