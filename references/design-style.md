# Reference: 设计审查（统一 · 简约 · 现代）

## 目标

用 lailai 的审美做 UI / 网页 / 平面 / 排版设计审查。提炼自站点 `docs/project/guides/design.mdx`（lailai 的《设计风格指南》，参考 Apple HIG、Material Design 3、WCAG）与 `.claude/CLAUDE.md` / `.github/CONTRIBUTING.md` 的工程约定。

## 适用场景

- 审查网页 / 组件 / 平面 / 排版设计是否符合"统一 · 简约 · 现代"。
- 给设计改进建议；实现 UI 时自检。

## 核心理念

**统一 · 简约 · 现代**（Unity · Simplicity · Modernity）。**统一优先于美观**。任何"局部更炫"若破坏统一，否决。

> lailai 自述："我不是专业设计师，所做的一切都基于直觉。" 审查时给判断和方向，不堆术语。

## 平面 / 排版规则（来自《设计风格指南》）

1. **统一**：设计风格保持统一，统一优先于美观。
2. **层级用视觉强度表达，而非复杂装饰**。
3. **颜色**：用柔和中性色 + 轻量强调色，避免饱和与冲突。**不用渐变色和半透明背景**。
4. **功能**：外观必须与功能匹配，界面忠于内容。设计不是展示，而是帮用户完成任务。
5. **文本**：遵循 [markdown-style.md](markdown-style.md) / [wording.md](wording.md)。小文字对比度 ≥ $4.5:1$，大文字 ≥ $3:1$。不孤字成行，不大面积留白。

## Web / 工程规则（来自站点约定）

6. **复用优先**：先用既有组件 / 原语（laikit：Card、Button、Badge、Segmented、Slider、Switch…），再考虑新造。新 UI 要和既有部分视觉与行为上看不出差别。
7. **绝不整卡 hover 上浮**：不给整张卡片 / 大容器加 hover 触发的 `translateY` / `translateX`。"卡片浮起"是 AI-tell，明确反感。小元素的克制动效（箭头轻移、图标 / 控件 `scale()`）服务清晰交互时可以。
8. **现代 CSS 优先**：`grid`、`clamp()`、`color-mix()`、容器查询优先于 JS 布局。
9. **系统字体优先**：SF Pro Text / Display、system-ui，辅以 Inter。
10. **颜色用 token**：主色干净（站点 `--ifm-color-primary: #1d9bf0`）；状态色走变量。不散落硬编码色值。
11. **CSS Modules**：组件 `styles.module.css` 与 `index.tsx` 同目录；全局 token 放 `custom.css`。
12. **深浅双主题必备**：任何有可视化 UI 的项目（web / app…）都必须同时支持深色与浅色主题，**默认跟随系统**（`prefers-color-scheme` / 系统设置），可再留手动切换。颜色全走 token、两套主题各定义一份，别只做一套硬塞。
13. **编辑而非重写**，改动最小，不重排无关样式。**精益求精**：间距、对齐、节奏、明暗、中英一致都要到位。

## 审查清单（打分时用）

1. **统一**：与既有组件 / 页面是否一致？复用了原语吗？
2. **简约**：有没有多余装饰、可删元素、过度动效、大面积留白？
3. **现代**：现代 CSS？字体 / 颜色 / 间距是否当代审美？
4. **AI-tell**：整卡上浮、渐变、半透明、霓虹、过饱和——有没有？
5. **层级与功能**：层级用视觉强度而非装饰？外观服务功能？
6. **可访问性**：对比度达标（4.5:1 / 3:1）？无孤字成行？
7. **细节**：间距、对齐、明暗、响应式、中英排版到位？

## 示例：审查意见

> 方向能用，但有几个 AI-tell 要先去掉，再谈细节。
>
> **必改**
>
> 1. 整卡 hover 上浮——AI-tell，去掉 `translateY`，改成箭头轻移或图标 `scale(1.05)`。
> 2. 渐变背景 + 半透明卡片——和"统一·简约"冲突，删掉，回到柔和中性纯色。
>
> **统一** 3. 三个相近蓝收敛成一个，走 `--ifm-color-primary`，别散落硬编码。4. 间距 `13/15/19` 不成体系，并到 4 的倍数刻度，用 token。
>
> **可访问性** 5. 暗色下小字对比不足 4.5:1，调深背景或提亮文字。
>
> 一句话：克制、统一、复用既有原语。

## Self-review Checklist

- [ ] 统一优先，复用了既有原语，新 UI 与既有看不出差别？
- [ ] 没有整卡 hover 上浮、渐变、半透明背景？
- [ ] 层级用视觉强度而非复杂装饰？柔和中性色 + 轻量强调色？
- [ ] 外观服务功能，不是为展示？
- [ ] 对比度 ≥ 4.5:1 / 3:1，无孤字成行、无大面积留白？
- [ ] 颜色 / 间距走 token，现代 CSS，系统字体？
- [ ] 深浅双主题都支持、默认跟随系统；改动最小，明暗 + 响应式都验证，细节到位？
