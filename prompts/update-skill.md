# Prompt: 根据新习惯更新 skill

复制以下内容给 AI，并描述新发现的习惯 / 偏好 / 反感项。

---

我发现了 lailai 的一个新习惯 / 偏好 / 反感项，请你按 lailai.skill 的维护流程决定如何处理，**不要直接乱改规则**。

新发现：

【在此描述：现象、场景、出现过几次、是否带原文证据】

请按以下流程处理：

1. **稳定性判断**：这是长期稳定习惯，还是短期情绪 / 一次性偏好？依据是什么？
   - 若不稳定或仅出现一次 → 只追加到 `observations.md`，不改正式规则。
2. **可沉淀性判断**（仅当稳定）：是否满足 稳定 / 可迁移 / 可执行 / 不涉隐私 / 非情绪？
3. **归档建议**：该进 `profile/` 的哪个文件，还是某个 `references/`？是长期规则还是局部规则（标明范围）？
4. **改动方案**：给出最小、精准的改动（编辑而非重写），并说明"为什么改"。
5. **连带更新**：是否需要更新对应 `examples/` 或 `tests/`？
6. **记录**：在 `CHANGELOG.md` 与 `evolution/evolution-log.md` 追加条目。
7. **人工确认**：若涉及长期人格 / 价值判断 / 隐私边界 / 公开表达，标注"需 lailai 确认"，先不落入正式规则。

原则：观察 ≠ 规则；临时情绪 ≠ 长期人格；复杂度服务准确性。若拿不准，宁可先放 `observations.md`。
