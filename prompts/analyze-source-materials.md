# Prompt: 分析资料、提炼风格

复制以下内容给 AI，并附上待分析的资料（文章、代码、README、设计、聊天样例）。

---

你在帮我维护 lailai.skill——一个复刻 lailai 表达 / 思维 / 审美 / 习惯的赛博分身系统。现在请你**分析我提供的资料，提炼可迁移的风格规则**，用女娲式蒸馏流程：

1. **收集**：通读全部资料，先不下结论。
2. **分类**：按维度归类——日常聊天 / 提问 / 写作 / 思考 / 判断 / 决策 / Markdown / Docusaurus / C++ / 设计 / 项目文档 / 学习。
3. **提炼模式**：找出**反复出现**的具体习惯（带原文证据 + 出处），不要泛泛而谈。
4. **交叉验证**：每条模式至少有 2 处证据才算"稳定"；只出现一次的标"待观察"。
5. **输出规则**：把稳定模式写成可执行规则（做 X、不做 Y），并区分：
   - 长期 / 可迁移规则（进 `profile/` 或通用 `references/`）；
   - 局部 / 项目规则（标明适用范围）。
6. **反例**：单独列出"不像 lailai"的表达，进 `ai-tone-blacklist.md`。

要求：

- 每条结论附具体证据（引用 + 出处），不编。
- 区分"稳定习惯"与"一次性现象"。
- 不要神化、不要玄学、不要空泛人设。
- 不收录隐私、密钥、他人敏感信息。
- 输出格式：按维度分节，每节列"规则 + 证据 + 建议归档位置 + 稳定性（稳定 / 待观察）"。
