# Prompt: 达尔文式优化 skill

复制以下内容给 AI，跑一轮"评估 → 改进 → 测试 → 保留或回滚"。

---

请用达尔文式流程优化 lailai.skill：每轮只改一个最弱点，验证后决定保留或回滚。

流程：

1. **评估**：用 `evolution/evaluation-rubric.md` 的 12 个维度，对当前 skill 的代表性输出打分（可用 `tests/` 里的任务生成输出）。列出各维度得分。
2. **定位最弱点**：找出最低分维度。若并列，选对"像不像 lailai"影响最大的。
3. **单点改进**：只针对这一个维度改进——改对应的 `profile/` 或 `references/`，给最小、精准的改动，说明假设（"这样改会让 X 维度从 a 升到 b，因为……"）。
4. **测试**：用对应的 `tests/` 重新生成输出并打分。
5. **保留或回滚**：
   - 目标维度变好且其它维度未明显变差 → 保留。
   - 目标维度没变好，或其它维度变差 → 回滚，记录失败假设。
6. **记录**：把本轮（触发原因、改动、前后得分、保留 / 回滚、是否需人工确认）写入 `evolution/evolution-log.md`，重要变化同步 `CHANGELOG.md`。

约束：

- 一轮只改一个主要问题，不要同时大改多处。
- 改动要可解释、可回滚。
- 复杂度必须服务准确性——若改进靠堆结构换来，质疑它。
- 涉及人格 / 价值 / 隐私 / 公开表达的改动，标注需人工确认。

输出：本轮评分表 → 最弱点 → 改动 → 复测评分 → 保留 / 回滚结论 → 日志条目。
