# Test: 设计审查

## 测试目标

验证 AI 的设计审查像 lailai：抓 AI-tell、强调统一与 token 化、给可落地改法。

## 输入任务

给一段 UI 描述（含若干问题），要求审查并给意见。测试样本：

> 列表卡片：hover 时整卡上浮 8px 加阴影；主色三种相近紫散落各处；圆角 `5px/8px/11px` 不一；背景用彩色渐变 + 发光边框；用 JS 监听 resize 算布局。

## 期望输出特征

- 必抓出"整卡 hover 上浮"并要求去掉，改为小元素克制动效。
- 抓出渐变 + 发光边框这类 AI-tell，要求回到克制方案。
- 指出颜色 / 圆角不统一，要求 token 化、并入刻度。
- 指出"JS 算布局"，建议改用现代 CSS（`clamp()` / 容器查询）。
- 先整体判断再逐条，每条给具体改法，语气直接。

## 评分标准（0～5）

- 0：没抓出整卡上浮，泛泛夸"挺好看"。
- 3：抓出部分问题，但改法笼统或漏掉 AI-tell。
- 5：抓全 AI-tell + 一致性 + 现代 CSS 问题，每条可落地。

## 常见失败表现

- 没识别整卡上浮是 AI-tell。
- 只说"建议统一颜色"不给 token 化做法。
- 意见抽象、不可执行。

## 客观检查项（可机械判定，全过才算通过）

- [ ] 明确指出"整卡 hover 上浮"是问题并要求去掉。
- [ ] 指出渐变 / 发光（半透明）背景应去掉。
- [ ] 指出颜色不统一并要求 token 化 / 收敛到一个主色。
- [ ] 指出圆角 / 间距不成体系并要求并入刻度。
- [ ] 指出"JS 算布局"应改用现代 CSS（`clamp()` / 容器查询）。
- [ ] 每条问题都给了具体改法，不是空泛"建议统一"。

## 通过条件

主观评分 ≥ 4 分，**且**上述客观检查项至少 5 项通过（"整卡 hover 上浮"为必含项）。
