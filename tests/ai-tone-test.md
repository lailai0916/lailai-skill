# Test: AI 腔识别与改写

## 测试目标

验证 AI 能识别 AI 腔并改写成 lailai 风格。

## 输入任务

给一段满是 AI 腔的文字，要求：(1) 指出所有 AI 腔问题；(2) 改写成 lailai 风格。

测试样本：

> 这是一个非常棒的问题！在当今数字化飞速发展的时代，掌握 Git 无疑是每位开发者的必备技能。接下来，就让我们一起深入浅出地揭开它神秘的面纱吧！总之，熟练运用 Git 将让你的开发效率如虎添翼，希望这篇文章能给你带来帮助哦～😊

## 期望输出特征

- 识别部分：逐条点出黑名单命中（套话开场、"在当今……时代"、"深入浅出 / 神秘面纱"、"总之"凑总结、"如虎添翼"、客套收尾、emoji），对照 [ai-tone-blacklist.md](../references/ai-tone-blacklist.md)。
- 改写部分：删全部套话，先结论后依据，给具体定义 / 用途，密度高、语气干。

## 评分标准（0～5）

- 0：没识别出问题，或改写后仍是 AI 腔。
- 3：识别出大部分，改写基本干净但残留 1～2 处。
- 5：识别全面准确，改写零 AI 腔、信息密度明显提高。

## 常见失败表现

- 只改了开场没改收尾。
- 改写后引入新套话（"简而言之""不难发现"）。
- 漏掉 emoji / 形容词堆叠。

## 客观检查项（可机械判定，全过才算通过）

- [ ] 识别部分至少点出 5 处黑名单命中（套话开场、"在当今…时代"、"深入浅出 / 神秘面纱"、"总之"凑总结、"如虎添翼"、客套收尾、emoji 任取 5）。
- [ ] 改写结果不含任何 [ai-tone-blacklist.md](../references/ai-tone-blacklist.md) 表格中的表达。
- [ ] 改写结果无 Emoji、无"显然 / 易得 / 不难发现"。
- [ ] 改写结果首句即给定义或结论。

## 通过条件

主观评分 ≥ 4 分，**且**上述客观检查项全部通过，且改写结果能通过 [writing-style-test.md](writing-style-test.md) 的客观检查项。
