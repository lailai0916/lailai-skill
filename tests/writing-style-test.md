# Test: 写作风格

## 测试目标

验证输出符合 lailai 中文写作风格：先结论后依据、结构化、高密度、无套话。

## 输入任务

让被测 AI 写一段 150～300 字的中文，解释"为什么递归要有终止条件"，面向有编程基础的读者。

## 期望输出特征

- 无套话开场 / 收尾。
- 先给结论（没有终止条件会无限递归 / 栈溢出），再给依据。
- 有定义或类比、有具体例子。
- 中英之间留空格，数量包进数学（若出现）。
- 标题与正文之间空行（若有标题）。
- 语气干、克制，无 emoji 堆叠。

## 评分标准（0～5）

- 0：满是套话、抒情、形容词堆叠，没有结论先行。
- 3：结构基本对，但有 1～2 处套话或密度偏低。
- 5：先结论后依据、结构清晰、有例子、零套话、密度高。

## 常见失败表现

- "在编程的世界里，递归是一种优雅的……"开场。
- "总之，希望这能帮助你理解递归～"收尾。
- 只讲概念不给例子。
- 中英不空格、句末标点进了行内代码。

## 客观检查项（可机械判定，全过才算通过）

- [ ] 不含「显然」「易得」「不难发现」「众所周知」。
- [ ] 不含「本文将」「希望」「帮助你」「综上所述」「总而言之」。
- [ ] 无 Emoji、无颜文字。
- [ ] 每句 ≤ 40 字（硬上限 60）。
- [ ] 中文与英文 / 数字之间有空格。
- [ ] 首句即给结论或定义（不是背景铺垫）。

## 通过条件

主观评分 ≥ 4 分，**且**上述客观检查项全部通过。
