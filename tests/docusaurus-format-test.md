# Test: Docusaurus 格式

## 测试目标

验证站内 MDX 题解 / 文章符合 lailai 的 Docusaurus 规范。

## 输入任务

为一道虚构题（PID `P9999`，求两数之和，洛谷）写一篇站内题解 MDX。

## 期望输出特征

- frontmatter：`title: 题解：P9999 <name>`、`tags: [oi, solution, luogu]`，字段顺序正确，`authors` 不写成数组（题解通常省 authors，按站点惯例 blog 默认作者亦可，至少不违规）。
- 第一行 `<Solution pid="P9999" aid="..." />`，紧跟 `{/* truncate */}`，前面无 prose。
- 正文从 `##` 起，无 H1。
- 数学用 `$...$`（不是 `\(...\)`），数量包进数学。
- `## 解题思路` 末尾有复杂度行，如 `时间复杂度为 $O(1)$。`。
- 代码块 `cpp`、无 fence title、无注释，遵循 C++ 模板。
- 无 `## 总结` / "希望帮到你"。

## 评分标准（0～5）

- 0：用 `\(...\)`、写了 H1、加了总结、frontmatter 乱、代码风格不符。
- 3：主体对，但有 1～2 处违规（如数学分隔符或缺复杂度行）。
- 5：模板、数学、代码、收尾全部合规。

## 常见失败表现

- 站内用了 `\(...\)` / `\[...\]`。
- 题解结尾加"总结 / 小结"。
- 代码带注释或用 `endl` / `scanf`。
- frontmatter 用了自造 tag。

## 客观检查项（可机械判定，全过才算通过）

- [ ] frontmatter 有 `title: 题解：…` 与 `tags: [oi, solution, luogu]`，无自造 tag。
- [ ] 第一行是 `<Solution .../>`，紧跟 `{/* truncate */}`。
- [ ] 正文无 `# `（一级标题），从 `##` 起。
- [ ] 数学用 `$...$` / `$$...$$`，不含 `\(` / `\[`。
- [ ] `## 解题思路` 段以 `时间复杂度为 $O(...)$。` 结尾。
- [ ] 代码块标签是 ` ```cpp `，块内无 `//` 注释、无 `endl`。
- [ ] 全文无「## 总结」「## 小结」「希望」。

## 通过条件

主观评分 ≥ 4 分，**且**上述客观检查项全部通过。
